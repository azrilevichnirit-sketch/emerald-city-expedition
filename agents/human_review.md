# Human Visual Review — בדיקת עין אנושית

## למה הסוכן הזה קיים

קוד יכול לבדוק שקובץ בשם `גלשן_מ01.png` קיים.
קוד לא יכול לבדוק שהתמונה בקובץ הזה היא אכן גלשן ולא משקפת.

זו הסיבה היחידה לסוכן הזה — **לראות בעיניים**.

---

## מתי הPipeline עוצר לבדיקה אנושית

### עצירה 1 — Style Test
אחרי שVisual Prompt Writer ייצר 3 כלים לבדיקה:

```
Pipeline עוצר.

הנה 3 תמונות לבדיקה:
[מציג: כלי A / כלי B / כלי C]

בדקי:
□ הסגנון מתאים לקהל 25+?
□ לא ילדותי?
□ ברור מה כל כלי?
□ תואם Camera Bible (זווית, תאורה)?

אישור? כן / לא + הערות
```

### עצירה 2 — כל כלי חדש שנוצר
לפני שכלי נכנס ל-asset_manifest:

```
Pipeline עוצר.

כלי חדש נוצר: [שם הכלי]
[מציג תמונה]

בדקי:
□ זה אכן [שם הכלי]? (לא כלי אחר)
□ רקע ירוק נקי — אין אלמנטים אחרים?
□ עקבי עם style_reference?
□ ברור מה הוא ואיך משתמשים בו?

אישור? כן / לא + מה הבעיה אם לא
```

### עצירה 3 — כל סצנה שנבנתה
אחרי Builder, לפני Content Validator:

```
Pipeline עוצר.

סצנה [ID] מוכנה לצפייה.
[קישור ל-output/scene_[ID].html]

פתחי בדפדפן ובדקי:
□ השחקנית נראית — אין ירוק?
□ הכלים ברורים ומזוהים?
□ כלי A הוא [שם] — זה מה שנראה?
□ כלי B הוא [שם] — זה מה שנראה?
□ כלי C הוא [שם] — זה מה שנראה?
□ הטקסט קריא?
□ נראה טוב על מובייל?

אישור? כן / לא + מה לתקן
```

---

## מה Producer עושה עם התשובה

**"כן"** → ממשיך לשלב הבא

**"לא + הערות"** → 
- בעיה בכלי (תמונה לא נכונה) → Visual Prompt Writer מחזיר לעבוד
- בעיה בסצנה → Builder מתקן לפי ההערות המדויקות
- בעיה בסגנון כללי → Production Designer מעדכן Camera Bible
